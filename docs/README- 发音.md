# 🔊 发音功能排查指南

如果点击按钮没有声音，请按以下步骤排查：

---

## 步骤 1：测试极简版页面

**打开文件**：`test-simple.html`（双击或用浏览器打开）

**操作**：
1. 等待 3 秒（让语音加载）
2. 点击「🔊 播放」按钮
3. 查看下方状态提示

**预期结果**：
- ✅ 显示「加载到 X 个语音」
- ✅ 显示「开始发音」
- ✅ 显示「发音完成」
- 🔊 能听到 "cat" 的发音

---

## 步骤 2：检查浏览器控制台

按 **F12** 打开控制台，查看：

### 正常情况
```
页面加载完成，正在加载语音...
✅ 加载到 10 个语音
准备播放：cat
使用语音：Google US English
已发送播放请求
▶ 开始发音
✅ 发音完成
```

### 异常情况
| 显示 | 原因 | 解决方法 |
|------|------|----------|
| ❌ 浏览器不支持 TTS | 浏览器太旧 | 换 Chrome/Edge |
| ⚠ 语音列表为空 | 语音未加载 | 刷新页面，等 3 秒再试 |
| ❌ 错误：not-allowed | 需要用户交互 | 先点击页面任意位置 |
| ❌ 错误：interrupted | 快速连续播放 | 等前一个说完再点 |

---

## 步骤 3：检查系统设置

### Windows
1. 右键点击任务栏喇叭图标
2. 打开「音量合成器」
3. 确认浏览器音量未静音

### macOS
1. 打开「系统偏好设置」→「声音」
2. 确认输出设备正确
3. 确认音量未静音

### 手机
- 按音量+键，确认**媒体音量**已打开（不是铃声音量）

---

## 步骤 4：浏览器兼容性

| 浏览器 | 支持情况 | 备注 |
|--------|----------|------|
| Chrome | ✅ 完美 | 推荐 |
| Edge | ✅ 完美 | 推荐 |
| Firefox | ✅ 支持 | 音质稍差 |
| Safari | ⚠ 需交互 | iOS 需先点击页面 |
| IE | ❌ 不支持 | 请换浏览器 |

---

## 步骤 5：修复主程序

如果 `test-simple.html` 能发音，但 `index.html` 不能：

**原因**：主程序语音加载时机问题

**解决**：在主程序打开后，先点击页面任意位置（激活音频上下文），再点击发音按钮。

---

## 常见解决方案

### 方案 1：刷新页面
最简单的方法，刷新后等 3 秒再试。

### 方案 2：更换浏览器
下载最新版 Chrome 或 Edge。

### 方案 3：检查权限
某些浏览器需要授予「自动播放」权限。

### 方案 4：手机特殊处理
iOS Safari 需要先与页面交互才能播放声音：
1. 点击页面任意位置
2. 再点击发音按钮

---

## 还是不行？

在浏览器控制台输入以下命令测试：

```javascript
// 测试 1：检查是否支持
'speechSynthesis' in window

// 测试 2：获取语音数量
speechSynthesis.getVoices().length

// 测试 3：直接发音
speechSynthesis.speak(new SpeechSynthesisUtterance('Hello World'))
```

如果测试 3 能发音，说明 TTS 正常，问题在代码逻辑。
如果测试 3 也不能发音，说明是浏览器或系统问题。

---

## 最终方案：使用外部音频文件

如果 TTS 始终无法使用，可以：

1. 录制 50 个单词的 MP3 文件
2. 放入 `audio/` 文件夹
3. 修改代码使用 `<audio>` 标签播放

免费 TTS 录音工具：
- https://ttsmaker.com/
- https://voicemaker.in/
