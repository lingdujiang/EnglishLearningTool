# 🔊 发音问题解决方案

## 问题诊断

根据测试，您的情况是：
- ✅ 浏览器支持 TTS（检测到 22 个语音）
- ✅ 音频输出正常（需要确认）
- ❌ 浏览器 TTS 无法发声（事件不触发）
- ❌ 在线 TTS 无法访问（网络限制）

## 🎯 推荐方案：使用本地 MP3 文件

这是**最可靠**的方案，不依赖网络和浏览器 TTS 引擎。

---

## 方案 A: 使用 Python 批量生成（推荐）

### 步骤 1: 安装 Python

如果还没安装，下载：https://www.python.org/downloads/

### 步骤 2: 安装 gTTS 库

打开命令提示符（CMD），输入：
```bash
pip install gTTS
```

### 步骤 3: 运行生成脚本

在文件夹中打开命令提示符，输入：
```bash
python generate-audio.py
```

脚本会自动：
- 创建 `audio/` 文件夹
- 生成 50 个 MP3 文件（cat.mp3, dog.mp3, ...）
- 每个文件约 10-20KB

### 步骤 4: 测试

打开 `index.html`，点击「🔊 听发音」按钮，应该能听到清晰的发音！

---

## 方案 B: 手动下载（如果没有 Python）

### 使用在线 TTS 工具

1. **TTSMaker** (推荐)
   - 网址：https://ttsmaker.com/
   - 选择 "English (US)" → "Google US English"
   - 输入单词（如 "cat"）
   - 点击「Convert」→ 下载 MP3
   - 重命名为 `cat.mp3`
   - 放入 `audio/` 文件夹

2. **重复 50 次**（可以分几天完成）

### 批量下载工具

**DownTTS** (GitHub):
- 网址：https://github.com/iterative/dvc.org
- 可以批量下载多个单词

## 方案 D: 修复浏览器 TTS（高级）

如果一定要用浏览器 TTS，尝试以下方法：

### Windows 系统

1. **检查 TTS 服务**
   ```
   Win + R → 输入 services.msc
   找到 "Speech Runtime Service"
   确保状态是"正在运行"
   ```

2. **安装语音包**
   ```
   设置 → 时间和语言 → 语音
   添加英语语音包
   ```

3. **测试系统 TTS**
   ```
   在语音设置页面，点击"预览"
   应该能听到测试声音
   ```

### Chrome 浏览器

1. **清除缓存**
   ```
   Ctrl + Shift + Delete
   清除缓存和 Cookie
   ```

2. **重置权限**
   ```
   chrome://settings/content/sound
   确保网站有声音权限
   ```

3. **更新浏览器**
   ```
   chrome://settings/help
   检查更新
   ```

## 常见问题

### Q: 生成的 MP3 文件播放不了？
A: 检查文件是否完整（应该>5KB），重新生成。

### Q: 点击发音按钮还是没声音？
A: 
1. 按 F12 打开控制台
2. 查看是否有错误信息
3. 确认 `audio/cat.mp3` 文件存在
4. 检查电脑音量和扬声器

### Q: 只想先测试几个单词？
A: 手动下载 5 个单词试试（cat, dog, bird, fish, rabbit），确认能发声后再批量生成。
